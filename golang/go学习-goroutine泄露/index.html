Go学习 Goroutine泄露

goroutine 泄露常见说明

参考文档

  • Go 笔记之如何防止 goroutine 泄露
  • Go 笔记之如何防止 goroutine 泄露（二）
  • Go 译文之如何构建并发 Pipeline

泄露情况分类

    1. 死循环导致goroutine 无法退出
    1. channel泄露
    • 发送无接收
    • 接收无发送
    • nil channel 发送和接收都会阻塞, 通过select方式, nil channel不会被select
    1. 传统同步机制
    • sync.Mutex 使用未释放, 建议Lock 后执行defer Unlock
    • sync.WaitGroup 设置的任务数错误导致一直 wait阻塞, 建议每次wg.Add(1)

简单实例


func Test4() {
	c := make(chan struct{})
	count := 0
	ctx, cancel := context.WithCancel(context.Background())
	timeout := time.After(time.Second * 2)

    go func() {
        for {
            select {
        	case c <- struct{}{}:
        		time.Sleep(time.Millisecond * 100)
        	case <-ctx.Done():
        		fmt.Println("停止写入 ... ")
        		return
        	}
        }
    }()
	for {
		select {
		case <-c:
			count++
		case <-timeout:
			fmt.Println("timeout ... ")
			return
		default:
			time.Sleep(time.Millisecond * 50)
			if count > 3 {
				fmt.Println("已经写入三次, 准备停止...")
				cancel()
				return
			}
		}
	}

}
